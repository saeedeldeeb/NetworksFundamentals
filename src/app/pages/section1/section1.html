Welcome to the Fundamentals of Network Engineering course

Lesson 1

Welcome

This course is designed for backend engineers, software engineers, and frontend developers who build network applications but lack a deep understanding of networking fundamentals.

Why This Course Exists

Many developers spend years building software without truly understanding:

  • What network stacks are and how they work
  • Current network limitations
  • TCP handshakes and acknowledgements
  • Why connections vary in speed

Traditional university courses often present networking concepts in isolation, without connecting them to real-world software development.

Course Goal

Bridge the gap between software engineering and network engineering by teaching first-principle building blocks.

Topics Covered

TCP/IP stack
UDP and TCP protocols
OSI layers (Layer 3, 4, 6, 7)
Essential concepts for backend & DevOps
Lesson 2

Who Is This Course For?

Backend Engineers

Those building backend applications that clients consume, but don't fully understand:

  • What happens when a segment arrives at their application
  • What the operating system does with incoming data
  • What the application is listening on
  • Why a segment goes to a certain process and not another
Key insight: The frontend consumes the backend, and the medium connecting them is networking — understanding this is fundamental.

Frontend Engineers

Those making API calls to backend services benefit from understanding:

  • How a request travels from client to server
  • Why slowdowns may not be the backend's fault
  • Low-level client-side network configurations
Key insight: When you understand networking, you know what to do. Without understanding, it becomes a black box.

Network Engineers

Those interested in building backend/frontend applications, or wanting to refresh their knowledge.

Key insight: There's a significant gap between network engineers and application developers. This course bridges that gap.
Lesson 3

Course Outline

The course is divided into six major sections, building from fundamentals to hands-on analysis.

1

Fundamentals of Networking

Why networking was invented, client-server architecture, host-to-host communication, and basic networking concepts from scratch.

2

Internet Protocol (IP)

IP address structure, packet routing, routing protocols, ARP, ICMP, and IP packet headers. "Don't memorize headers — understand what each does."

3

UDP (User Datagram Protocol)

UDP datagram structure, pros and cons, when to use and when not to use UDP. Starting with UDP because it's simpler than TCP.

4

TCP (Transmission Control Protocol)

TCP segments, three-way handshake, flow control, congestion control, slow start, TCP Fast Open, and connection states. "Anything exists because of a problem it solves."

5

Networking & Backend Performance

The bridge section — linking network engineering to backend/frontend performance. MSS, MTU, packet sizes, and HTTP request behavior.

6

Wireshark Analysis

Hands-on packet analysis: sniffing packets, analyzing TCP/UDP/TLS protocols, and decrypting TLS handshakes.
