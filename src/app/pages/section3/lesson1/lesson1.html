DHCP — How Devices Get an IP Automatically

When you connect to a network, your device doesn't magically know its IP address. DHCP (Dynamic Host Configuration Protocol) handles this through a 4-step handshake known as DORA:

D
Discover Your device broadcasts: "I'm new here, I need an IP address!" Since it has no IP yet, it sends to the broadcast address 255.255.255.255.
O
Offer The DHCP server responds: "How about 192.168.1.50? It's available." The server picks an IP from its pool and reserves it temporarily.
R
Request Your device confirms: "Yes, I'll take 192.168.1.50!" This is broadcast so other DHCP servers (if any) know the offer was accepted.
A
Acknowledge Server finalizes: "Confirmed! Here's your IP, subnet mask, default gateway, DNS server, and lease time." Your device is now fully configured.
What DHCP Gives You
IP Address — Your identity on the network Subnet Mask — Defines your local network boundary Default Gateway — The router to reach other networks DNS Server — Translates domain names to IPs Lease Time — How long you keep this IP (must renew)
DHCP vs Static
DHCP Automatic, convenient, IPs can change. Used by phones, laptops, IoT devices.
Static Manual, fixed IP. Used for servers, printers, routers — things that need a stable address.

IP Address Structure

Every IP has two parts: a network portion and a host portion, divided by the CIDR notation.

Example: 192.168.254.0/24
192 . 168 . 254 Network (24 bits)
.0 – .255 Host (8 bits)
224 ~16M possible networks
254 usable hosts per network

Subnet and Subnet Mask

A subnet is a logical subdivision of an IP network. The subnet mask determines which part is network vs host.

CIDRSubnet MaskHost BitsMax Hosts
/8 255.0.0.0 24 ~16 million
/16 255.255.0.0 16 65,534
/24 255.255.255.0 8 254
/32 255.255.255.255 0 1 (single host)

Default Gateway

Every network needs a default gateway — a router with multiple network interfaces, living in multiple networks at once.

Three Things Every Host Needs

1 IP Address
2 Subnet Mask
3 Default Gateway

Without these three, you're "dead in the water."

Security Note: ARP Poisoning

When sending to the gateway, you use ARP to find its MAC address. An attacker can pretend to be the router — all your packets then flow through the attacker instead (Man-in-the-Middle).

Backend Performance Insight

Database in different subnet

Every query goes through the router. If the router is congested, your SQL statements wait in its buffer. Result: unexplained latency.

→
Database in same subnet

Use a dedicated switch (Layer 2). Switches are designed for high-throughput local traffic — no routing overhead.
