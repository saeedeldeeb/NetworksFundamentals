The IP Packet

The anatomy of the IP packet — understanding the structure beyond just source and destination IP.

IP Packet Structure

Every IP packet has two sections: a header (metadata) and a data section (payload).

Header
20–60 bytes
Version, TTL, Protocol, Source/Dest IP, Flags...
Data
Up to 65,536 bytes
The actual payload (TCP segment, UDP datagram, etc.)

Why Header Size Matters

20 B header 1 B
Sending 1 byte = 21 bytes total (95% overhead!)
20 B 1,480 B payload
Sending 1,480 bytes = 1,500 bytes total (1.3% overhead)

This is why Nagle's Algorithm exists — it batches tiny writes to avoid sending many small packets with huge relative overhead.

IP Header Layout

The IP header is read in 32-bit (4-byte) rows. Minimum 5 rows = 20 bytes.

Time to Live (TTL)

Problem: IP is stateless — routers don't remember packets. Without TTL, a packet could loop forever between misconfigured routers.

If TTL reaches 0, the packet is dropped and the router sends an ICMP "Time Exceeded" message back to the source. This is how traceroute works — by sending packets with incrementally increasing TTLs.

Fragmentation

When a packet is larger than the MTU (Maximum Transmission Unit, typically 1,500 bytes), it must be fragmented or dropped.

Modern approach: Most protocols (like QUIC) disable fragmentation and use Path MTU Discovery instead.

Protocol Field

Tells routers what's inside the data section — so they can make decisions from just the 20-byte header.

1 ICMP Diagnostics & errors
6 TCP Reliable transport
17 UDP Fast, unreliable transport
41 IPv6 IPv6 encapsulation
50 ESP IPsec encryption

Explicit Congestion Notification (ECN)

A smarter way to handle network congestion — without dropping packets.

IP Spoofing

Can you fake the source IP? Technically yes — but practically it rarely works.

1
You write a fake source IP Packet leaves with spoofed address
2
ISP checks source IP ISP assigned your real IP — they know it's fake. Packet dropped.
3
Even if it gets through... Response goes to the fake IP, not you. You never see the reply.
For backend engineers: The IP packet is usually just "data with source and destination IP." But understanding headers helps debug network issues — TTL for traceroute, fragmentation for performance, ECN for congestion.
