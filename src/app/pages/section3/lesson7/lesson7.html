Private IP Address Ranges

These IPs are reserved for local networks. They are not routable on the public internet — packets with these addresses will be dropped.

RangeCIDRHost BitsTotal

Real Example: Airplane WiFi

Alaska Airlines WiFi Configuration

IP 172.19.31.48
Mask 255.255.254.0 (/23)
GW 172.19.30.1
Why /23 instead of /24?
/24 256 addresses (254 usable)
/23 512 addresses (510 usable)
~200 passengers × 2-3 devices = need 512+
Port-Based Service Filtering
Allowed HTTP (80), DNS (53), text messages
Blocked HTTPS (443), images, voice calls
Router blocks ports for bandwidth control

Private vs Public IP

Private IP
Internet Not routable
NAT Required to access internet
Reusable? Yes, in different networks
Public IP
Internet Globally routable
NAT Not needed
Reusable? No, globally unique
Key takeaway: Private IPs require NAT. Subnet mask determines capacity — one extra bit doubles hosts. Real networks are designed based on device count, and port filtering enables granular access control.
