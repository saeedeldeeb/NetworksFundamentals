Internet Protocol (IP)

Deep dive into IP addressing, packets, ICMP, ARP, and routing

Lesson 1

The IP Building Blocks

Fundamental concepts needed to understand the IP protocol — addressing, subnets, and gateways.

IP Address Basics

Layer 3 (Network)
Size 4 bytes (32 bits)
Assignment DHCP or Static
Format A.B.C.D/X

DHCP — How Devices Get an IP Automatically

When you connect to a network, your device doesn't magically know its IP address. DHCP (Dynamic Host Configuration Protocol) handles this through a 4-step handshake known as DORA:

D
Discover Your device broadcasts: "I'm new here, I need an IP address!" Since it has no IP yet, it sends to the broadcast address 255.255.255.255.
O
Offer The DHCP server responds: "How about 192.168.1.50? It's available." The server picks an IP from its pool and reserves it temporarily.
R
Request Your device confirms: "Yes, I'll take 192.168.1.50!" This is broadcast so other DHCP servers (if any) know the offer was accepted.
A
Acknowledge Server finalizes: "Confirmed! Here's your IP, subnet mask, default gateway, DNS server, and lease time." Your device is now fully configured.
What DHCP Gives You
IP Address — Your identity on the network Subnet Mask — Defines your local network boundary Default Gateway — The router to reach other networks DNS Server — Translates domain names to IPs Lease Time — How long you keep this IP (must renew)
DHCP vs Static
DHCP Automatic, convenient, IPs can change. Used by phones, laptops, IoT devices.
Static Manual, fixed IP. Used for servers, printers, routers — things that need a stable address.

IP Address Structure

Every IP has two parts: a network portion and a host portion, divided by the CIDR notation.

Example: 192.168.254.0/24
192 . 168 . 254 Network (24 bits)
.0 – .255 Host (8 bits)
224 ~16M possible networks
254 usable hosts per network

Subnet and Subnet Mask

A subnet is a logical subdivision of an IP network. The subnet mask determines which part is network vs host.

CIDRSubnet MaskHost BitsMax Hosts
/8 255.0.0.0 24 ~16 million
/16 255.255.0.0 16 65,534
/24 255.255.255.0 8 254
/32 255.255.255.255 0 1 (single host)

How Subnet Mask Works

Apply a bitwise AND between the IP and subnet mask to determine which network a host belongs to. Watch it alternate between same-subnet and different-subnet examples:

The Decision Tree

Is destination in my subnet?
YES

Send directly using MAC address (Layer 2)

NO

Send to default gateway (router)

Default Gateway

Every network needs a default gateway — a router with multiple network interfaces, living in multiple networks at once.

Three Things Every Host Needs

1 IP Address
2 Subnet Mask
3 Default Gateway

Without these three, you're "dead in the water."

Security Note: ARP Poisoning

When sending to the gateway, you use ARP to find its MAC address. An attacker can pretend to be the router — all your packets then flow through the attacker instead (Man-in-the-Middle).

Backend Performance Insight

Database in different subnet

Every query goes through the router. If the router is congested, your SQL statements wait in its buffer. Result: unexplained latency.

→
Database in same subnet

Use a dedicated switch (Layer 2). Switches are designed for high-throughput local traffic — no routing overhead.

Key insight: Understanding networking helps you ask the right questions to network engineers. They know everything about configuration — but they need you to tell them what you need.
Lesson 2

The IP Packet

The anatomy of the IP packet — understanding the structure beyond just source and destination IP.

IP Packet Structure

Every IP packet has two sections: a header (metadata) and a data section (payload).

Header
20–60 bytes
Version, TTL, Protocol, Source/Dest IP, Flags...
Data
Up to 65,536 bytes
The actual payload (TCP segment, UDP datagram, etc.)

Why Header Size Matters

20 B header 1 B
Sending 1 byte = 21 bytes total (95% overhead!)
20 B 1,480 B payload
Sending 1,480 bytes = 1,500 bytes total (1.3% overhead)

This is why Nagle's Algorithm exists — it batches tiny writes to avoid sending many small packets with huge relative overhead.

IP Header Layout

The IP header is read in 32-bit (4-byte) rows. Minimum 5 rows = 20 bytes.

Time to Live (TTL)

Problem: IP is stateless — routers don't remember packets. Without TTL, a packet could loop forever between misconfigured routers.

If TTL reaches 0, the packet is dropped and the router sends an ICMP "Time Exceeded" message back to the source. This is how traceroute works — by sending packets with incrementally increasing TTLs.

Fragmentation

When a packet is larger than the MTU (Maximum Transmission Unit, typically 1,500 bytes), it must be fragmented or dropped.

Original Packet (4,000 bytes)
↓ MTU = 1,500 bytes
Fragment 1
Offset 0, MF=1
Fragment 2
Offset 1480, MF=1
Fragment 3
Offset 2960, MF=0
DF=0 (Allow Fragment)
Action Split into smaller fragments
Receiver Must reassemble
Slower, but packet gets through
DF=1 (Don't Fragment)

Response ICMP "Fragmentation Needed"
Packet lost, sender adjusts MTU

Modern approach: Most protocols (like QUIC) disable fragmentation and use Path MTU Discovery instead.

Protocol Field

Tells routers what's inside the data section — so they can make decisions from just the 20-byte header.

1 ICMP Diagnostics & errors
6 TCP Reliable transport
17 UDP Fast, unreliable transport
41 IPv6 IPv6 encapsulation
50 ESP IPsec encryption

Explicit Congestion Notification (ECN)

A smarter way to handle network congestion — without dropping packets.

IP Spoofing

Can you fake the source IP? Technically yes — but practically it rarely works.

1
You write a fake source IP Packet leaves with spoofed address
2
ISP checks source IP ISP assigned your real IP — they know it's fake. Packet dropped.
3
Even if it gets through... Response goes to the fake IP, not you. You never see the reply.
For backend engineers: The IP packet is usually just "data with source and destination IP." But understanding headers helps debug network issues — TTL for traceroute, fragmentation for performance, ECN for congestion.
Lesson 3

ICMP, Ping, and Traceroute

ICMP (Internet Control Message Protocol) is a critical Layer 3 protocol used for network diagnostics and error reporting.

What is ICMP?

Layer 3 (Network)
Uses IP directly, no ports
Protocol # 1
Listener None needed

Common ICMP Messages

TypeMessageWhen Sent
0Echo ReplyResponse to ping
3Destination UnreachableHost/port unreachable
5RedirectBetter route available
8Echo RequestPing request
11Time ExceededTTL reached 0

Ping

Sends an ICMP Echo Request and waits for an Echo Reply. Tests if a host is reachable and measures round-trip time.

Why ICMP Gets Blocked

Some firewalls block ICMP because of flooding attacks, network probing by attackers, and backchannel attacks using ICMP for covert communication.

TCP Black Hole (ICMP Blocking Problem)

Key insight: ICMP is essential infrastructure. Blocking it can cause hard-to-debug network issues like TCP black holes. When ping doesn't work, don't assume the host is down — ICMP might be blocked.
Lesson 4

ARP (Address Resolution Protocol)

ARP maps IP addresses to MAC addresses — essential for sending frames on a local network.

The Problem ARP Solves

What We Know
Hostname Use DNS → get IP address
IP Address Use ARP → get MAC address
ARP bridges Layer 3 (IP) and Layer 2 (MAC)
Why We Need MAC
Frames Require MAC for delivery
Switches Use MAC to forward frames
IP alone isn't enough for local delivery

How ARP Works

Host A wants to send to Host B. It broadcasts "Who has this IP?" to all devices. Only Host B replies with its MAC address. A caches the result in its ARP table.

ARP Poisoning Attack

ARP has no authentication. Anyone can answer ARP requests.

1
Host A broadcasts: "Who has 10.0.0.1 (gateway)?" Normal ARP request to find the router
2
Attacker responds: "That's me!" Sends fake ARP reply with attacker's MAC address
3
All traffic goes through attacker Man-in-the-Middle: Host A → Attacker → Gateway → Internet

VRRP (Virtual Router Redundancy)

ARP enables high availability: multiple routers share one virtual IP. If the primary fails, the backup starts answering ARP requests — hosts don't notice.

For backend engineers: Understanding ARP explains why the first packet to a new host is slower (ARP lookup), how MITM attacks work on local networks, and how high-availability setups like keepalived work.

Practical packet analysis using tcpdump — a command-line packet analyzer available on macOS and Linux.

Common Options

Quick Reference

tcpdump -i en0 -n arp
Capture all ARP packets
tcpdump -i en0 -n icmp
Capture all ICMP (ping)
tcpdump -i en0 -n host 1.2.3.4
Filter by specific host
tcpdump -i en0 -n -v icmp
Verbose ICMP details
tcpdump -i en0 -w file.pcap
Save for Wireshark
tcpdump -n icmp and host 8.8.8.8
Combine filters

Key Observations

1 ARP is Layer 2 — just MAC addresses
2 ICMP is Layer 3 — source/dest IP, no ports
3 Ports appear at Layer 4 (TCP/UDP)
Key insight: At the IP layer, there are no ports — only source and destination IP addresses. Ports come with TCP/UDP in the transport layer.

Routing Example

How IP packets flow through switches and routers — a practical walkthrough of same-network, cross-network, and internet routing.

Layer 2 vs Layer 3 Destinations

MAC Address (Layer 2)
Scope Local delivery only
Changes? Yes — at every hop
Switch uses MAC to forward frames
IP Address (Layer 3)
Scope End-to-end routing
Changes? No — stays constant

Key takeaway: Same network = switch + MAC. Different network = router. Internet = router + NAT. IP destination stays constant; MAC destination changes per hop.
