Section 04

UDP (User Datagram Protocol)

Understanding the simplest Layer 4 protocol — stateless, connectionless, and fast

Lesson 2

UDP Datagram Structure

The anatomy of the UDP datagram — understanding the 8-byte header that makes UDP the simplest transport protocol.

How UDP Fits Inside IP

A UDP datagram slides into an IP packet as its data portion. The IP header's protocol field is set to 17, telling the receiver "this data is UDP." The IP packet doesn't care what's inside — it just sees bytes.

The UDP Header

Only 4 fields, each 16 bits (2 bytes). Two rows of 32 bits each give us the 8-byte header. Compare that to TCP's 20–60 bytes — UDP keeps it minimal.

Source & Destination Ports

Both are 16 bits, giving a range of 0–65,535. Ports identify which application on a host should receive the datagram.

Port Service Purpose
53 DNS Domain name resolution
67/68 DHCP Automatic IP assignment
69 TFTP Simple file transfer
123 NTP Time synchronization
161 SNMP Network monitoring
500 IKE VPN key exchange

Port Exhaustion

If you fix the destination IP, destination port, and source IP, then only the source port varies — limiting you to 65,535 concurrent connections to the same server.

Fixed
Dst IP: 8.8.8.8 Dst Port: 53 Src IP: 10.0.0.1
Variable
Src Port: 1024 ... 65535
Max ~65,535 connections

Length Field

Specifies the total size of the UDP datagram (header + data). At 16 bits, the maximum value is 65,535 bytes. Since the header is always 8 bytes, the minimum value is 8 (header only, no data).

Minimum
Header 8 bytes
Data 0 bytes
Length = 8
Maximum
Header 8 bytes
Data 65,527 bytes
Length = 65,535

Checksum Field

The checksum detects data corruption during transmission. Electrical interference, voltage attenuation, or even cosmic rays can flip bits in transit.

IPv4 vs IPv6: In IPv4, the UDP checksum is optional. In IPv6, it's mandatory — because IPv6 removed the IP-level checksum, so the transport layer must ensure integrity.

UDP vs TCP Header

Field UDP TCP
Source Port 16 bits 16 bits
Destination Port 16 bits 16 bits
Sequence Number — 32 bits
Acknowledgment — 32 bits
Flags — 9 bits
Window Size — 16 bits
Checksum 16 bits 16 bits
Length / Urgent Ptr 16 bits 16 bits
Total Minimum 8 bytes 20 bytes
Key insight: UDP's 8-byte header carries only what's essential — ports for addressing, length for sizing, and a checksum for integrity. No sequence numbers, no acknowledgments, no flow control. This simplicity means more bandwidth for your actual data and less processing at each hop.
