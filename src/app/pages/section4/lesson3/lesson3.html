Understanding the simplest Layer 4 protocol — stateless, connectionless, and fast

Lesson 3

UDP Pros & Cons

Everything has tradeoffs. Understanding UDP's strengths and weaknesses helps you choose the right protocol for the job.

Pros of UDP

⚡
Simple Protocol
Does exactly what developers need — nothing more. No unnecessary overhead. Perfect when you want control over the communication logic.
📦
Small Header (8 bytes)
More bandwidth available for actual data. Less processing at each network hop. Faster packet handling.
📈
Stateless & Scalable
No memory consumed for connection tracking. Server can handle many more concurrent clients. Process and forget.
🚀
Low Latency
No handshake, no acknowledgments, no retransmission delays. First packet = your actual data.

Latency Comparison

TCP (4 packets before data)
Pkt 1 SYN
Pkt 2 SYN-ACK
Pkt 3 ACK
Pkt 4 DATA (finally!)
1.5 round trips before data
UDP (1 packet)
Pkt 1 DATA (immediately!)
Zero round trips — instant

Stateful vs Stateless

TCP: Each Connection Stores
Memory Window sizes, sequences, buffers, timers
Scale Memory grows with each connection
1000s of connections = significant RAM
UDP: Nothing Stored
Memory Process and forget — zero state
Scale Theoretically infinite clients
Scales with almost no overhead

Cons of UDP

1
No Acknowledgment Sender doesn't know if data arrived. Can't verify the server received or processed anything. Bad for databases, financial transactions, or any critical data.
2
No Guaranteed Delivery Packets may be lost or arrive out of order. No automatic retransmission. The application must handle losses itself.
3
Connectionless (Security Risk) No authentication between parties. Anyone can send to anyone. Source addresses are easily spoofed — enabling reflection attacks.
4
No Flow Control Sender has no idea how much the receiver can handle. Can overwhelm the receiver's buffer. Packets dropped silently with no feedback.
5
No Congestion Control No awareness of network conditions. Can contribute to congestion. Routers may drop packets silently with no backoff mechanism.
6
No Packet Ordering Packets may arrive out of sequence. The application must reorder them if order matters — or accept out-of-order data.

UDP-Based Attacks

UDP's connectionless, stateless nature makes it vulnerable to abuse. Two common attacks exploit these weaknesses:

DNS Amplification Attack

The attacker sends small DNS queries to many DNS servers but forges the source IP to be the victim's address. DNS servers send their large responses to the victim, flooding it with unsolicited traffic. A 40-byte query can produce a 3,000-byte response — a 75x amplification.

UDP Flood Attack

The attacker sends massive amounts of UDP datagrams to a server. Since UDP is stateless, the server must process every single packet — it can't filter by connection state like TCP can. This consumes CPU and memory until the server is overwhelmed.

Building Custom Reliability on UDP

Some developers choose UDP and build only the features they need. This gives maximum control — pick TCP-like features without the full cost.

Full TCP Features
Order Full sequence tracking
Reliability Every packet acknowledged
Flow Window-based flow control
Congestion Slow start, avoidance algorithms
All features, all overhead
Custom on UDP (e.g. Game)
Order Simple sequence numbers
Reliability Selective — only critical events
Flow Simple rate limiting
Congestion Not needed
Only what you need, minimal overhead
Key insight: UDP gives you a blank canvas. TCP gives you a complete painting. Choose based on whether you need the full picture or want to paint your own.
