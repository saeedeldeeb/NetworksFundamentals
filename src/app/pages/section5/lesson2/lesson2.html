Section 05

TCP (Transmission Control Protocol)

The backbone of reliable internet communication — stateful, connection-oriented, and controlled

Lesson 2

TCP Segment

The anatomy of a TCP segment — understanding the 20-byte header and the fields that make reliability possible.

How TCP Fits Inside IP

A TCP segment slides into an IP packet as its data portion. The IP header's protocol field is set to 6 (vs UDP's 17), telling the receiver "this data is TCP."

The TCP Header

The header has a minimum of 20 bytes (5 rows of 32 bits) and can grow up to 60 bytes with options. Compare to UDP's 8 bytes — TCP's complexity comes at a cost.

Header Fields Explained

Source & Destination Ports

16 bits each (same as UDP), range 0–65,535. Identifies which application sends and receives — exactly like UDP.

Sequence Number

32 bits Sequence Number
Tracks the order of data sent — increments with each byte
Range: 0 to ~4 billion, then wraps back to zero
Same wraparound problem exists in PostgreSQL transaction IDs

Acknowledgment Number

32 bits Acknowledgment Number
Only valid when the ACK flag is set
Tells sender: "I've received everything up to this number"
Separate from sequence — you can send data AND acknowledge simultaneously

Window Size (Flow Control)

16 bits Window Size
Default max: 65,535 bytes (seems small for modern networks)
Tells sender: "This is how much data I can receive right now"
Window Scale option (negotiated during handshake) can increase this to ~1 GB

The 9 Flags

Each flag is a single bit that controls different aspects of the connection. They're grouped into three categories:

Connection
SYNInitiate connection (handshake)
FINClose connection gracefully
RSTAbort connection immediately
Data
ACKAcknowledge received data
PSHDeliver to app immediately
URGData is urgent (rarely used)
Congestion
ECERouter signaled congestion
CWRSender reduced sending rate
NSECN nonce protection

How Congestion Flags Work

Instead of waiting for packet loss to detect congestion, ECN lets routers warn senders early — no data is lost, and the sender slows down proactively.

Maximum Segment Size (MSS)

How much data can fit in one TCP segment? It depends on the MTU (Maximum Transmission Unit) — the Layer 2 limit.

MTU (Ethernet) 1500 bytes
IP Header − 20 bytes
TCP Header − 20 bytes
MSS = 1460 bytes

If your data exceeds the MSS, TCP splits it into multiple segments automatically. Some networks support jumbo frames (MTU ~9000 bytes, MSS ~8960 bytes) — common inside cloud provider networks for better performance.

Key insight: TCP's 20-byte header (vs UDP's 8 bytes) carries all the machinery for reliability: sequence numbers for ordering, acknowledgments for confirmation, window size for flow control, and flags for connection management. This overhead is the price of reliability.
