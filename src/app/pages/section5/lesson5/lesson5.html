Lesson 5

Slow Start vs Congestion Avoidance (Deep Dive)

A detailed look at how TCP's two congestion control algorithms work together — probing the network aggressively at first, then carefully, and resetting when things go wrong.

The Two Algorithms

TCP uses two distinct algorithms for congestion control. The slow start threshold (ssthresh) determines which one is active:

if cwnd < ssthresh: Slow Start Exponential growth
if cwnd ≥ ssthresh: Congestion Avoidance Linear growth

Slow Start: The Details

Why "slow"? We start from 1 MSS — a very slow beginning. Why "fast"? We increase aggressively — exponentially.

The rule: For every ACK received, add 1 MSS to cwnd.

1
Send 1 → 1 ACK → +1 → cwnd = 2
2
Send 2 → 2 ACKs → +2 → cwnd = 4
3
Send 4 → 4 ACKs → +4 → cwnd = 8
4
Send 8 → 8 ACKs → +8 → cwnd = 16

Every single acknowledgment adds 1 MSS. If you send 10 packets and get 10 ACKs, you add 10 MSS. This is what makes growth exponential — more packets means more ACKs means faster growth.

Congestion Avoidance: The Details

After cwnd reaches ssthresh, growth slows dramatically. The rule: Add 1 MSS per complete round trip, not per ACK.

What Is a Round Trip in TCP?

Send cwnd worth of segments, then wait for ACKs for all of them. Only when all are acknowledged = 1 complete round trip.

Application-level RTT: send request → get response. TCP-level RTT: send cwnd segments → get all ACKs back.

Example: cwnd = 4 segments

ACK 1 ACK 2 ACK 3 ACK 4
All 4 acknowledged = 1 RTT → cwnd = 5 (+1 only)

Congestion Detection

How does TCP know there's congestion? It uses a retransmission timer:

1 Send a segment, start timer
2 ACK arrives before timer expires → OK
3 Timer expires before ACK → Packet lost = Congestion assumed

The assumption: routers in the middle dropped the packet because their buffers overflowed.

What Happens When Congestion Is Detected

Two things happen:

1 ssthresh = flight_size / 2 (halve the in-flight data)
2 cwnd = 1 MSS (reset to slow start)

flight_size vs cwnd

A subtle but important distinction — the new threshold is based on flight_size, not cwnd:

cwnd
Maximum you can send
≠
flight_size
What you actually have in flight (unacknowledged)

Why the Distinction Matters

cwnd 10 MSS (you can send up to 10)
flight_size 3 MSS (only 3 unacknowledged)
Wrong ssthresh = 10 / 2 = 5
Right ssthresh = 3 / 2 ≈ 2 MSS

Minimum Threshold

Can ssthresh keep decreasing to zero? No. There's a floor:

ssthresh = max(flight_size / 2, 2 × MSS)
floor The slow start threshold can never go below 2 MSS

The Full Cycle

Watch the complete congestion control cycle: slow start ramps up exponentially until ssthresh, then congestion avoidance grows linearly. When packet loss is detected, cwnd crashes to 1 and the cycle restarts — but with a lower ssthresh each time.

Reading the Chart

Each cycle through the chart reveals a pattern: the ssthresh keeps decreasing. You reach it faster, so you spend less time in slow start and more time in congestion avoidance. Eventually ssthresh hits the minimum of 2 MSS.

Impact on Your Application

Before loss cwnd = 1000 segments — sending fast
After loss cwnd = 1 segment — sending slow

Your app feels this. Downloads slow down, connections feel sluggish. Then it ramps back up through slow start and congestion avoidance — the sawtooth pattern repeats.

Fast Retransmit

There's another algorithm called Fast Retransmit that detects congestion faster using duplicate ACKs instead of waiting for timeout. It's less harsh — doesn't reset cwnd to 1. Not covered in detail here, but worth knowing it exists.

These algorithms are defined in RFC 5681 (TCP Congestion Control). The HOMA paper proposes reimagining TCP for data centers, redefining what congestion means — worth reading after understanding these fundamentals.

Key insight: The congestion control algorithm is a continuous dance — aggressively ramping up to find capacity, then backing off when hitting limits. Your application's perceived speed depends heavily on this cycle. Understanding it helps you diagnose why connections start slow, speed up, then occasionally stutter.
