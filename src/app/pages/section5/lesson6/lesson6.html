Lesson 6

NAT (Network Address Translation)

How billions of devices share the internet with only 4 billion IPv4 addresses. NAT is the brilliant hack that extended IPv4's life by decades.

The Problem

IPv4 gives us ~4 billion addresses (32 bits = 232). There are far more devices than that. Two solutions exist:

IPv6
128-bit addresses — virtually unlimited
Adoption is slow
NAT
Share one public IP among many private devices
Used everywhere

Private IP Ranges

Three address ranges are reserved for private networks. They are not routable on the internet — routers drop them.

10.0.0.0/8 10.0.0.0 — 10.255.255.255 16M addresses
172.16.0.0/12 172.16.0.0 — 172.31.255.255 1M addresses
192.168.0.0/16 192.168.0.0 — 192.168.255.255 65K addresses

How NAT Works

Your entire household shares one public IP. The router rewrites packet headers on the way out and reverses the translation on the way back.

The Translation Process

Outgoing
1
Laptop sends packet
Src 192.168.1.20:8992 → Dst 55.11.22.33:443
2
Router rewrites source
Src 44.11.5.17:7777 → Dst 55.11.22.33:443
Private IP replaced with router's public IP + random port
NAT Table Entry Created
192.168.1.20:8992 ↔ 44.11.5.17:7777 → 55.11.22.33:443
Return
3
Google responds to the router
Dst 44.11.5.17:7777
Google has no idea a private device exists
4
Router translates back
Dst 192.168.1.20:8992
Looks up NAT table, restores original private address

NAT Is Stateful

The router must remember every active connection. Different devices and ports create separate entries:

router# show nat translations
PRIVATEPUBLICDESTINATION
192.168.1.20:8992 44.11.5.17:7777 55.11.22.33:443
192.168.1.20:9001 44.11.5.17:7778 55.11.22.33:443
192.168.1.30:5555 44.11.5.17:7779 88.99.11.22:80
192.168.1.10:4444 44.11.5.17:7780 55.11.22.33:443
4 active translations

NAT Operates at Layer 3 AND Layer 4

A normal router only reads IP addresses (Layer 3). A NAT router must also rewrite ports (Layer 4) — making it significantly more complex.

Feature Normal Router NAT Router
IP addresses Reads Reads AND rewrites
Layers Layer 3 only Layer 3 + Layer 4
Ports Doesn't touch Rewrites too

Connection Limits

With source IP, destination IP, and destination port fixed, only the source port varies — capping you at ~65,535 connections to the same server. These ports are shared across all devices on your network.

NAT Applications

🌐
Private → Public The original purpose — billions of private devices share limited public IPv4 addresses
🔌
Port Forwarding Map external port 80 → internal 192.168.1.50:8080. Run servers as non-root, expose publicly on well-known ports
⚖
Layer 4 Load Balancing Rewrite destination IP to distribute traffic across servers (round robin). Used by HAProxy. Client never knows which server handled the request

Limitations

Port exhaustion Only 65,535 ports per public IP per destination
Stateful overhead Must track every active connection in memory
Breaks end-to-end Server can't initiate a connection to the client
P2P issues Both sides behind NAT = complex traversal needed
Key insight: NAT is a brilliant hack that extended IPv4's life by decades. It lets your entire household share one public IP, enables port forwarding for servers, and even powers some load balancing solutions. The tradeoff is complexity — NAT routers must track state and operate at multiple layers, and some protocols (like peer-to-peer) struggle with NAT's one-way nature.
